# 前言《量潮大模型选型报告》写法分析

对同一篇前言的两次分析，分工不同：

| 文件 | 分析对象 | 提问 |
|------|---------|------|
| `foreword.md` | 前言文本本身 | 它是什么结构？每一段在做什么？意图是什么？ |
| `author.md` | 写前言的人 | 从措辞和行文来看，创作者是什么性格？ |

## 对 Agent 写作的指导

一、**自己买单** — 写判断要带依据、带数据、带事实，不接受空泛的「我觉得」。

二、**把牌亮在桌上** — 前提、立场、意图先说清楚，不让人在字缝里猜。

三、**不要装** — 去掉「让我看起来很专业」的废话，只留最有信息量的部分。

四、**画面感优先** — 「一个任务就够 DeepSeek 烧一个下午了」比「贵了 30 倍」有力得多。
